# CLAUDE.md

This repository follows the AI Project Documentation Policy.

## Core Principle

> The AI proposes. The human decides. The repository records.

## Mandatory Behavior

When contributing to this repository, you must:

1. Follow existing project conventions and architecture.
2. Update `progress.md` after significant milestones.
3. Record design rationale in `decisions.md`.
4. Create entries in `ai_journal/YYYY-MM-DD.md` when AI materially influences outcomes.
5. Save high-impact prompts in `prompts/`.
6. Create ADRs in `adr/` for major architecture decisions.
7. Write modular, tested, and well-documented code.
8. Explicitly state assumptions, uncertainties, and recommended verification steps.
9. Never treat unverified output as authoritative.

## AI Contribution Levels

0 = No AI used
1 = Brainstorming only
2 = AI proposed approach
3 = AI generated substantial draft code/text
4 = AI produced most of the artifact

## Preferred Workflow

- Understand the problem.
- Propose solutions.
- Explain trade-offs.
- Support implementation.
- Recommend tests.
- Record decisions and AI influence.

## Documentation Priority

Documentation is part of the deliverable, not an afterthought.
