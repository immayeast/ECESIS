- Treat bus mapping as graph-aware entity resolution rather than plain fuzzy matching.
- Use latitude/longitude as a first-stage candidate-generation strategy, not as the sole match criterion.
- Keep nearest-neighbor fallbacks and name/voltage fallbacks for missing-coordinate cases.
- Add conservative name normalization while preserving numeric identifiers and voltage-related information.
- Build Dayzer and Panorama branch graphs separately, with buses as nodes and branches as edges.
- Use local graph fingerprints such as degree, neighbor voltage profile, neighbor token overlap, and topology score.
- Combine name, voltage, geographic, and topology evidence into an interpretable weighted composite score.
- Preserve component features for auditability and manual review.
- Flag ambiguous matches using score gaps, evidence conflicts, and one-to-one duplicate Panorama claims.
- Use high-confidence seed matches and topology-aware propagation as a future refinement path for missing-coordinate or weak-name cases.
- Write the final report as a concise infrastructure graph matching study instead of a notebook dump.
- Make the final strategy hierarchy explicit: seeds first, topology as the main refinement signal, geography as validation, voltage as physical feasibility, one-to-one conflict resolution, and manual review.
- Add an iterative topology-aware propagation stage that accepts candidates supported by already-matched neighbors.
- Evaluate graph-aware matching using internal consistency diagnostics because no labeled ground truth is available.
